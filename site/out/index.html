Everything after generation

Six modules that turn raw AI output into finished, platform-ready content.

B

Creative Brief

Standardized intake for brand, audience, and platform targets.

P

Provider Broker

Route stills, video, and utilities to the right provider.

S

Brand Presets

Treat style, formatting, and safe-area rules as code.

E

Export Profiles

Package outputs for TikTok, Reels, Shorts, YouTube, 1:1, 16:9, and 9:16.

Q

QA / Eval

Catch caption, aspect-ratio, brand, and output-coverage issues before publish.

H

Publish Handoff

Deliver scheduler-ready, platform-ready assets and metadata.

Provider-agnostic by design

Generation providers create the assets. OpenAI multimodal evaluates and reviews them. The broker decides where each job belongs based on modality and quality tier.

broker.route({
  type: "video.generate.premium",
  // → fal / kling-video/v3/pro
});

broker.route({
  type: "still.generate.fast",
  // → fal / nano-banana-2
});

Brand presets as code

Fonts, colors, watermarks, title-safe rules, aspect ratios, subtitle styles, and export rules are versioned configuration — not tribal knowledge.

{
  "brand": "acme-studio",
  "fonts": { "heading": "Inter", "body": "DM Sans" },
  "colors": { "primary": "#1868db", "bg": "#ffffff" },
  "safe_area": { "top": 64, "bottom": 80 },
  "watermark": { "position": "bottom-right", "opacity": 0.4 }
}

Publish-ready, not just model-ready

The bottleneck is not raw generation. It is everything after: packaging, formatting, QA, and publish handoff. This pipeline closes the gap.

pipeline.run(brief)
  .generate()    // fal / Gemini / Imagen
  .polish()      // caption cleanup, upscale
  .brandCheck()  // preset compliance
  .export([      // platform packages
    "tiktok-9x16",
    "reels-9x16",
    "youtube-16x9",
  ])
  .evaluate()    // QA gate
  .publish();    // handoff

FAQ

Is this a creator app?+

No. It is the infrastructure layer between generation and publishing. You bring your own UI or orchestration.

Does it replace fal, Gemini, Imagen, or OpenAI?+

No. It brokers across them. Each provider handles what it does best — the kit normalizes the workflow around them.

Is it focused on stills only?+

No. The model map includes stills, video, audio, utility transforms (background removal, upscaling, vector conversion), and more.

Is this for agencies only?+

No. It is for agency UGC, studio workflows, creator pipelines, and any team that needs a repeatable mediaops workflow.

Why open-source this?+

Because the finishing/export/QA layer is still thin or proprietary across the current landscape. No open-source tool covers it.
